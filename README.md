# Plan w/ AI

**Do Caos ao Kanban** — paste a messy list of tasks, let Gemini turn it into an organized Kanban board, and persist the boards in Firestore.

This Flutter app is a port of the Angular reference project [`refs/workshop_gdg`](refs/workshop_gdg/), keeping the same class names (`App`, `Quadro`, `Tarefa`, `GeminiService`, `QuadrosService`) and folder structure under `lib/`. It runs on **Web, Android, and iOS**.

## Stack

- **Flutter** 3.41.x / Dart 3.11.x
- **Gemini** via [`google_generative_ai`](https://pub.dev/packages/google_generative_ai) (`gemini-2.5-flash`)
- **Cloud Firestore** via [`firebase_core`](https://pub.dev/packages/firebase_core) + [`cloud_firestore`](https://pub.dev/packages/cloud_firestore)

## How it works

1. The user pastes a messy list of tasks into a textarea.
2. `GeminiService.organizarTarefas` sends a structured prompt (in Portuguese) to `gemini-2.5-flash` asking it to return a JSON `Quadro` (board) with categorized `Tarefa` items (title, category, priority, effort, suggested deadline, status).
3. The board is rendered as a 3-column Kanban (A fazer / Em andamento / Concluído). Cards can be moved between columns locally.
4. `QuadrosService.salvarQuadro` writes the current board to the `quadros` Firestore collection with an ISO `criadoEm` timestamp.
5. `QuadrosService.listarQuadros` loads the history (ordered by `criadoEm` desc) and shows it under the board.

## First-time setup

### 1. Install dependencies

```bash
flutter pub get
```

### 2. Provide your Gemini API key

The Gemini key lives in `lib/environments/environment.dart`, which is **gitignored** so secrets never leave your machine.

```bash
cp lib/environments/environment.example.dart lib/environments/environment.dart
```

Then open `lib/environments/environment.dart` and replace `YOUR_GEMINI_API_KEY_HERE` with a key from [Google AI Studio](https://aistudio.google.com/app/apikey).

### 3. Wire up Firebase

`lib/firebase_options.dart`, `android/app/google-services.json`, and `ios/Runner/GoogleService-Info.plist` are also gitignored. Generate them locally with the FlutterFire CLI:

```bash
dart pub global activate flutterfire_cli
flutterfire configure --project=workshop-gdg-rec --platforms=android,ios,web --yes
```

Accept the default Android package name and iOS bundle ID (`com.example.plan_w_ai`).

> If you don't have access to the `workshop-gdg-rec` Firebase project, pass your own project ID instead. Make sure the Firestore database exists and a `quadros` collection is writeable by your Security Rules.

### 4. (iOS only) Install CocoaPods

```bash
cd ios && pod install && cd ..
```

## Running the app

```bash
# Web (Chrome)
flutter run -d chrome

# Android
flutter run -d <android-device-id>

# iOS
flutter run -d <ios-device-id>
```

List available devices with `flutter devices`.

## Tests

```bash
flutter test
```

Covers:

- `App` widget construction (smoke test mirroring `app.spec.ts`).
- `Tarefa` and `Quadro` JSON roundtrips — pins the Angular union-type strings (`a_fazer`, `em_andamento`, `esta_semana`, `concluido`, etc.) that Gemini emits.

## Reference Angular project

The original Angular implementation lives under [`refs/workshop_gdg/`](refs/workshop_gdg/). Every Dart file in `lib/` corresponds 1:1 with a TypeScript file there:

| Angular (`refs/workshop_gdg/src`) | Flutter (`lib`) |
|---|---|
| `main.ts` | `main.dart` |
| `app/app.config.ts` | `app/app_config.dart` |
| `app/app.routes.ts` | `app/app_routes.dart` |
| `app/app.ts` + `app.html` + `app.css` | `app/app.dart` |
| `app/models/quadro.model.ts` | `app/models/quadro_model.dart` |
| `app/models/tarefa.model.ts` | `app/models/tarefa_model.dart` |
| `app/services/gemini.service.ts` | `app/services/gemini_service.dart` |
| `app/services/quadros.service.ts` | `app/services/quadros_service.dart` |
| `environments/environment.ts` | `environments/environment.dart` + `firebase_options.dart` |

The Gemini prompt in `GeminiService` is copied verbatim from the Angular original so both apps produce identical board structures.
